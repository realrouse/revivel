# Changelog — ReviveL v0.5.0

**Date:** 2026-07-15  
**Previous release:** 0.4.4  
**Scope:** Major feature release (Channels, Upload, multi-media playback, publish/Companion pipeline)

---

## Summary

0.5.0 turns ReviveL from a primarily browse/play extension into a fuller LBRY client surface inside the browser. Users get dedicated **Channels** and **Upload** flows in both the popup and the full player, improved **Library** organization, **multi-type media** playback (video / image / audio / other), and a more reliable path from **right-click → stage → publish** via the Companion and local daemon.

---

## Added

### Channels
- New **Channels** tab in the popup and full player.
- **Top Channels** list (ordered by effective stake / support).
- **My Channels** list (wallet-owned channels via `channel_list` / Companion).
- Infinite scroll / load-more for top channels in the popup.
- Channel claim badges and compact row layout (thumbnail + title/meta left, **Channel** badge right-aligned).
- Opening a channel can route into search / channel content browsing in the player.

### Upload & publishing
- New **Upload** tab in popup and player with sub-views:
  - **Upload content** (title, description, bid, optional channel, file picker)
  - **My Channels** (owned channels)
  - **Create channel** (`channel_create` path)
- **Right-click Quick Upload** (context menu “Upload image to ReviveL”):
  - **ON (default):** automatic publish/save using filename as title, fixed description, Anonymous channel
  - **OFF:** opens Upload tab with the image prefilled for manual editing
- Settings toggle for Quick Upload (popup + player settings).
- File staging helper (`stageFile`) for Companion-backed publishes.
- Background support for `getChannels`, `getMyClaims`, improved `publish` handling (`stream_create` vs `channel_create`, param sanitization).
- Native messaging bridge message type `sendNativeMessage` for Companion RPC.

### Player / media
- Renamed playback path to **`loadMedia`** (was video-centric).
- Native UI elements for **image** and **audio** claims; non-media files show download/open UI.
- Content-type detection from claim stream metadata (`stream_type` / `media_type`).

### Library
- Library sub-tabs: **Recent**, **My Uploads**, **Saved**.
- My Uploads can merge local library entries with on-chain claims from the wallet when available.

### Thumbnails & UI helpers
- Shared `createThumbHtml` helper: real thumbs, deterministic color placeholders, optional Channel badge overlay.
- Additional spacing / sub-tab styles shared between popup and player.
- Popup content padding and layout polish for denser forms (wallet, upload, settings).

---

## Changed

- Quick Upload default **stake/bid** set to **`0.0001` LBC** (was effectively dust-level `0.00000001`, which failed network/dust limits).
- Publish parameter cleanup: strip invalid stream fields (`file_type`, `file_name`, `file_size`) that could break lbrynet SDK.
- Wallet transaction listing path improved (richer mapping of txo/history fields, pagination params).
- Channel list rows force **row** flex layout so the badge sits on the right instead of stacking under content (override of `.compact-list-item` column default).
- Various wallet UI / link coloring fixes carried from the post-0.4.4 fix commit.

---

## Fixed

- Channel list items appearing too tall after badge placement (column flex + centered stack).
- Publish failures related to unsupported stream_create fields.
- Wallet transaction display gaps and link contrast issues.
- Feed infinite scroll reliability improvements from the 0.4.x line remain in place as baseline.

## Compatibility

- **Public / no-daemon mode:** browse, resolve, and play via public proxy still work; full publish and wallet still require Companion / local lbrynet.
- **Companion required** for real on-chain publish, channel_create, wallet balance/send, and staged file paths.
- Manifest V3; Chrome / Chromium-based browsers primary target.
